# Web GUI

Browser-based interface for managing the full pipeline — trigger actions, watch live logs, browse the database, edit config.

## Start

### Docker
```bash
docker compose up -d
# Open http://localhost:3721
# NAS: http://<nas-ip>:3721
```

### Local
```bash
pip install flask gunicorn
python -m web.app
# Open http://localhost:5000
```

## Pages

### Dashboard

Shows database stats (pending/scraped/error counts per site), a quick-actions panel with the most common pipeline commands, and recent action history. Each action shows live streaming output via SSE — you see scraper progress or audit results in real time.

### Actions

Trigger any pipeline step with parameters:
- **Ingest** — source dir, dry-run, auto-scrape, auto-enrich
- **Scrape FC2 / JavDB** — IDs, retry errors, flagged only, delay, dry-run
- **Enrich FC2 / JavDB** — IDs, dry-run
- **Flag** — mark entries for re-scrape by CID
- **Reorganize** — dry-run, IDs, structure report
- **Audit** — type filter (FC2/JAV), IDs, dry-run
- **Setup** — first-run initialization

Output streams live via Server-Sent Events (SSE). One action at a time — the single gunicorn worker prevents concurrent Playwright/Chromium instances.

### Database

Browse FC2 and JAV entries with:
- **Filters**: status (pending/scraped/error/404/flagged), site source
- **Search**: by CID or title
- **Sort**: click column headers
- **Expand**: click a row to see all fields (title, seller, studio, tags, duration, cover URL, etc.)
- **Flag**: mark entries for re-scrape directly from the table
- **Pagination**: 50 entries per page

### Logs

View dated report files generated by each pipeline step:
- Ingest reports (`ingest-YYYY-MM-DD.md`)
- Reorganize reports (`reorganize-fc2-YYYY-MM-DD.md`, `reorganize-jav-YYYY-MM-DD.md`)
- Duration audit reports (`duration-audit-YYYY-MM-DD.md`)
- Structure reports (`reorganized-structure-YYYY-MM-DD.md`)

Reports are rendered as HTML from Markdown (tables, code blocks supported).

### Config

Full YAML config editor with syntax highlighting (CodeMirror 5). Edit paths, cookies, thresholds, templates, and studio maps. On save:
- YAML syntax is validated — errors are shown inline, invalid config is rejected
- Previous config is backed up to `config.yaml.bak`

## Architecture

```
Flask (single worker)
├── /                Dashboard + stat cards
├── /actions         Action trigger UI
├── /db              Database browser
├── /logs            Report viewer
├── /config          YAML config editor
├── /api/action/*    Trigger pipeline steps → SSE stream output
├── /api/db/*        Database queries (JSON)
├── /api/config      Config read/write (JSON)
└── /api/report/*    Report file viewer (HTML)
```

One gunicorn worker with threading enabled — safe for Playwright's single Chromium instance. SSE streams subprocess stdout/stderr to the browser in real time.
